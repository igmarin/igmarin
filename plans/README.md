## Guiding Principles

1. **Revenue before architecture.** No new features until validation confirms demand.
2. **rails-ai-bridge is the center.** Everything else is content, evaluation, or archived.
3. **ruby-skill-bench is the differentiator.** Measurable quality is the moat.
4. **Orchestrator skills, not agents.** Deterministic workflows are honest. Real agents come later.
5. **Talk to humans before writing code.** Phase 0 validation is mandatory.
